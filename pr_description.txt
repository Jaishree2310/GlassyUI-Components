# Feat: Advanced Implementation of Modular Glassmorphic Checkbox System

## Overview
This PR addresses the "ENHANCEMENT: Advanced Implementation of Modular Glassmorphic Checkbox System" issue. 

Previously, the `Checkbox` component in this library was a basic, incomplete placeholder that did not align with the premium glassmorphism design language seen in our other advanced components (like Buttons, Modals, and Sliders). It lacked essential enterprise features like grouped state management, indeterminate capabilities, and accessibility handling.

This enhancement completely replaces the placeholder with a production-ready, modular, and highly customizable Checkbox system. It introduces a dedicated Context-driven wrapper for handling array-based multi-select logic and implements a scalable SCSS architecture that perfectly captures the modern UI aesthetic of GlassyUI-Components.

## Features Implemented
* **Controlled & Uncontrolled Support:** Seamless handling for both single booleans and array-based logic.
* **Indeterminate State:** Direct DOM-level mapping for parent-child "Select All" checkbox trees.
* **Grouped Checkbox Support:** Introduced `<CheckboxGroup>` for managing an array of selected options automatically.
* **Size & Color Variants:** Highly customizable predefined sizes (`sm`, `md`, `lg`) and theme variants (`primary`, `success`, `warning`, `danger`, `info`).
* **Glassmorphism UI:** Translucent backgrounds, layered blur effects, and neon accents.
* **Animations:** Smooth SVG `stroke-dashoffset` transitions for checkmarks and premium hover micro-interactions.
* **Accessibility First:** Full keyboard navigation, `aria-checked`/`aria-invalid` mapping, and focus-visible rings.
* **Responsive Design:** Scales flawlessly across mobile and desktop interfaces.
* **Modular SCSS:** Reusable variables, mixins, and dynamic state-classes.
* **Dark Mode Compatibility:** Beautiful rendering across varying application themes.

## Technical Improvements
* **Reusable Architecture:** Separated concerns by migrating from a single monolith file to a modular folder structure (`src/components/Checkbox/*`).
* **TypeScript Typing:** Strict and scalable interfaces (`CheckboxProps`, `CheckboxGroupProps`) heavily mirroring enterprise libraries like Radix and Shadcn.
* **State Handling Improvements:** Context API (`useCheckboxGroup`) prevents prop-drilling and cleanly manages array-state updates via custom utility logic.
* **Maintainability:** Abstracted array toggling and tailwind class merging into a lightweight `utils.ts` layer.

## Accessibility Enhancements
* **Semantic HTML:** Utilizes native `<input type="checkbox">` wrapped semantically in `<label>` elements.
* **ARIA Support:** Safely casts dynamic `aria-invalid` and `aria-checked` attributes (including the `"mixed"` state) to support screen readers securely.
* **Keyboard Support:** Fully operable via `Tab` and `Space` with a customized, glowing focus-visible ring for visual tracking.
* **Contrast Compliance:** SCSS variables ensure appropriate text and border contrast over frosted glass backgrounds.

## UI/UX Enhancements
* **Glassmorphism Effects:** Achieved via stacked `-webkit-backdrop-filter` rules, translucent RGBA borders, and sleek inner shadows.
* **Neon Glow Borders:** Active and focused states emit a subtle, neon-style `box-shadow` glow matching futuristic design aesthetics.
* **Micro-animations:** Checking the box triggers a literal "drawing" animation of the SVG checkmark for a deeply satisfying tactile feel.
